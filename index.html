Technical Interview Questions

HTML

Structure & Semantics

Basic HTML Concepts +
Q: What is HTML and what does it stand for?

HTML stands for HyperText Markup Language. It's the standard markup language used to create web pages and web applications.

Q: Explain the difference between HTML elements and tags.

HTML tags are the markup that define elements. An element includes the opening tag, content, and closing tag. For example: <p>Content</p>

Semantic HTML +
Q: What is semantic HTML and why is it important?

Semantic HTML uses meaningful tags that describe content structure (like <header>, <nav>, <article>). It improves accessibility, SEO, and code maintainability.

Q: Name some semantic HTML5 elements.

<header>, <nav>, <main>, <article>, <section>, <aside>, <footer>, <figure>, <figcaption>

CSS

Styling & Layout

CSS Fundamentals +
Q: What is CSS and what does it stand for?

CSS stands for Cascading Style Sheets. It's used to style and layout web pages, controlling colors, fonts, spacing, and positioning.

Q: Explain the CSS box model.

The box model consists of content, padding, border, and margin. Each element is treated as a rectangular box with these layers.

Flexbox & Grid +
Q: What is Flexbox and when would you use it?

Flexbox is a one-dimensional layout method for arranging items in rows or columns. It's great for component layouts and centering content.

Q: Difference between CSS Grid and Flexbox?

Grid is two-dimensional (rows and columns), Flexbox is one-dimensional. Use Grid for page layouts, Flexbox for component layouts.

JavaScript

Logic & Interactivity

JavaScript Fundamentals +
Q: What is JavaScript and what can it do?

JavaScript is a programming language that enables dynamic content, interactive features, and complex functionality in web browsers and servers.

Q: Explain the difference between let, const, and var.

var: function-scoped, can be redeclared. let: block-scoped, can be reassigned. const: block-scoped, cannot be reassigned.

Asynchronous JavaScript +
Q: What are Promises in JavaScript?

Promises are objects representing the eventual completion or failure of an asynchronous operation, providing a cleaner alternative to callbacks.

Q: Explain async/await syntax.

async/await provides a more readable way to work with Promises. async functions return Promises, await pauses execution until Promise resolves.
